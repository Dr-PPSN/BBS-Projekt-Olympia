inwiefern es möglich ist, den Stack auch für lokale Benutzung als Container anzubieten.

# Grundlegender Aufbau von Environments

Ein Spack Environment wird durch eine spack.yaml beschrieben. Diese beinhaltet im einfachsten Fall eine Liste von Packeten (keyword specs), die das Environment enthalten soll. Dazu sollte man konfigurieren, wie die Packete gebaut werden sollen (keyword packages) und
wie die Specs konkretisiert werden sollen (default ist ...). Der Großteil dieser Config Files kann auch ausgelagert werden, sodass man zum Updaten des Stacks nur eine übersichtlichere Datei verändern muss.
Die Vorteile von Environments gegenüber dem Bauen und Laden von einzelnen Spack Modulen sind, dass Environments konsistenter sind, mehrere Packete auf einmal gebaut werden können, und das gesamte Environment auf einmal geladen werden kann.

Ablauf von Environment erstellen
- erfolgt in zwei Schritten:
    - spack create env spack.yaml  -> erstellt ein Environment aus der spack.yaml
    - spack install                -> konkretisiert und installiert die Packete
